
### Listing process

The `ps` command stands for "process snapshot" or "process status". It lists the processes. By default, it list the processes *currently running*. 

```bash
hacker@dojo:~$ ps
    PID TTY          TIME CMD
    329 pts/0    00:00:00 bash
    349 pts/0    00:00:00 ps
hacker@dojo:~$
```

As you can see, the `bash` and `ps` process itself that is running on that specific terminal. Each process has a numerical identifier called the *Process ID* or *PID*, a unique identifier for every process. We can also see in the terminal on which the commands are running (the `pts/0) and the total amount of cpu time they eaten up so far (they are under 1 second since the process mentioned are very undemanding).

You can enrich `ps` even more with the following:

- In Standard Syntax, `-e` means to list every process and `-f` means to output the full format in the terminal, including the arguments. This will leave us with the combined argument `-ef`.
- In BSD Syntax, use `a` to list process for all users, `x` for process not running in a terminal, and `u` for user-readable output. Combining all of this will lead us to combined single argument `aux`.

`ps -ef` and `ps aux` outputs slightly different but cross-recognizable output.

### Killing Processes

You can also terminate processes in Linux. This is done using `kill` command. Let's say you have a running `sleep` process that you launched in another terminal:

```bash
hacker@dojo:~$ sleep 1337
```

You can use `kill` to terminate it by passing the PID of the `sleep` process as an argument:

```bash
hacker@dojo:~$ ps -e | grep sleep
 342 pts/0    00:00:00 sleep
hacker@dojo:~$ kill 342
hacker@dojo:~$ ps -e | grep sleep
hacker@dojo:~$
```

### Interrupting Processes

Aside from `kill` command, you sometimes want to kill the process that is currently clogging up your terminal. You can use hotkey for this without typing a command: `Ctrl-C` hotkey sends an interrupt to whatever the application is waiting on input from the terminal and causes the application to cleanly exit (typically).

### Suspending Processes 

Sometimes, you want to stop the process to clog your terminal, and resume them at some point. For this one, you can suspend the process in the background. Use `Ctrl-Z` hotkey to suspend the process in the background, allowing you to use the terminal again.

### Resuming Process

You want to resume suspended processes at some point. To resume process, use the `fg` command. `fg` is a builtin that takes the suspended process, resumes it, and puts it back in the foreground of your terminal.

### Backgrounding Process

With `fg` command, we can resume process in the foreground. But we can also resume process in the background with `bg` command, allowing the process to keep running, while giving you the ability to invoke more commands in the same terminal.

For demonstration, let' suspend`sleep`:

```bash
hacker@dojo:~$ sleep 1337
^Z
[1]+  Stopped                 sleep 1337
hacker@dojo:~$
```

The `sleep` process is now suspended in the background. Let's use `ps` and enable the `stat` column using argument `-o`:

```bash
hacker@dojo:~$ ps -o user,pid,stat,cmd
USER         PID STAT CMD
hacker       702 Ss   bash
hacker       762 T    sleep 1337
hacker       782 R+   ps -o user,pid,stat,cmd
hacker@dojo:~$ 
```

In PID 762, we can see that `sleep` is running. In the `STAT` column, `T` means the process is suspended due to `Ctrl-Z`. The `S` in `bash`'s `STAT` column means that `bash` is sleeping while waiting for input. `R` in `ps`'s column means that it is actively running, and the `+` means that it is in the foreground.

When we resume `sleep` in the background using `bg`:

```bash
hacker@dojo:~$ bg
[1]+ sleep 1337 &
hacker@dojo:~$ ps -o user,pid,stat,cmd
USER         PID STAT CMD
hacker       702 Ss   bash
hacker       762 S    sleep 1337
hacker      1224 R+   ps -o user,pid,stat,cmd
hacker@dojo:~$
```

Notice the `sleep` command now has an `S`. It's sleeping, but it's not suspended! It's also not in the background thus doesn't have the `+`.

### Directly background processes without suspending them

You don't have to suspend the process to background them: you can background them right away by appending `&` to the end of the command:

```bash
hacker@dojo:~$ sleep 1337 &
[1] 1771
hacker@dojo:~$ ps -o user,pid,stat,cmd
USER         PID STAT CMD
hacker      1709 Ss   bash
hacker      1771 S    sleep 1337
hacker      1782 R+   ps -o user,pid,stat,cmd
hacker@dojo:~$ 
```

Here, `sleep` is actively running in the background, not suspended.

### Process exit codes

Every shell command, including every program and builtin, exits with an *exit code* after it finishes running and terminates. You can check the exit code of the most recently-terminated command using `?` variable.

```bash
hacker@dojo:~$ touch test-file
hacker@dojo:~$ echo $?
0
hacker@dojo:~$ touch /test-file
touch: cannot touch '/test-file': Permission denied
hacker@dojo:~$ echo $?
1
hacker@dojo:~$
```

As you can see, commands that succeed typically return `0` and return a non-zero when it fail typically, with`1` as most common value. Sometimes, an error code identifies a specific failure mode.
