### `tr` command

`tr` command translates characters it receives over standard input and prints them to standard output. It can also handle multiple characters with different positions in the first argument replaced with associated characters in the second argument.

````bash
hacker@dojo:~$ echo OWN | tr O P
PWN
hacker@dojo:~$
````

````bash
hacker@dojo:~$ echo PWM.COLLAGE | tr MA NE
PWN.COLLEGE
hacker@dojo:~$
````

using `-d` flag translate characters to nothing (i.e. delete them)

````bash
hacker@dojo:~$ echo PAWN | tr -d A
PWN
hacker@dojo:~$
````

you can also use escape characters (\\). For example, replacing underscore with newline:

````bash
hacker@dojo:~$ echo "hello_world!" | tr _ "\n"
hello
world!
hacker@dojo:~$
````

to delete newlines:

````bash
mayo@localhost:~> cat message    
yo  
hello world  
mayo@localhost:~> cat message | tr -d "\n"  
yohello worldmayo@localhost:~>
````

### `head` command

Sometimes, file can be big and `cat`ing them can populate the terminal aggressively. `head` command display the first few lines of input. By default, it displays the first 10 lines but you can control the number of lines to display using `-n` flag.

````bash
hacker@dojo:~$ cat /something/very/long | head
this
is
just
the
first
ten
lines
of
the
file
hacker@dojo:~$
````

````bash
hacker@dojo:~$ cat /something/very/long | head -n 2
this
is
hacker@dojo:~$
````

### `cut` command

Sometimes, you want to grab specific columns of data in the file. For example, grabbing only the first column, or second column, or third column, etc. `cut` command lets you do that:

````bash
hacker@dojo:~$ cat scores.txt
hacker 78 99 67
root 92 43 89
hacker@dojo:~$
````

To use `cut`:

````bash
hacker@dojo:~$ cut -d " " -f 1 scores.txt
hacker
root
hacker@dojo:~$ cut -d " " -f 2 scores.txt
78
92
hacker@dojo:~$ cut -d " " -f 3 scores.txt
99
43
hacker@dojo:~$
````

The `-d` argument specifies the delimiter (space in this case), and the `-f` argument specifies the *field* number you want to extract.

### `sort` command

Files (or output lines of commands) aren't always in the order you need them! The `sort` command helps you organize data. It reads lines from input (or files) and outputs them in sorted order:

````bash
hacker@dojo:~$ cat names.txt
  hack
  the
  planet
  with
  pwn
  college
hacker@dojo:~$ sort names.txt
  college
  hack
  planet
  pwn
  the
  with
hacker@dojo:~$
````

By default, `sort` orders lines alphabetically. Arguments can change this:

* `-r`: reverse order (Z to A)
* `-n`: numeric sort (for numbers)
* `-u`: unique lines only (remove duplicates)
* `-R`: random order!
