
### File Descriptor Numbers

File Descriptor (FD) is a number that describes a communication channel in Linux. Here are the most common three:

- FD 0: Standard Input (`stdin`)
- FD 1: Standard Output (`stdout`)
- FD 2: Standard Error (`stderr`)
### left caret (>)

"\>" symbol means to redirect the `stdout` to a destination (usually a file). A single \> means to replace the entire content of file with the redirected output or to create a new file and append the redirected output. A double \> means to append the redirected output instead of overwriting the file.

- \> by default redirects `stdout`, but it's explicit form is `<stdout> 1> <file_dest\`.  To redirect `stderr`, use `2>`

```console
mayo@localhost:~> echo "hello, world" > my_file  
mayo@localhost:~> cat ./not_exist 2>> my_file  
mayo@localhost:~> cat my_file    
hello, world  
cat: ./not_exist: No such file or directory  
mayo@localhost:~>
```

You can redirect multiple file descriptors at the same time. For example:

```console
hacker@dojo:~$ some_command > output.log 2> errors.log
```

### right caret (<)

Just like the left caret, (>), the right caret (<) redirects `stdin` from a file to a program

```console
mayo@localhost:~> cat < my_file    
hello, world  
cat: ./not_exist: No such file or directory
```


### The pipe (|)

The pipe operator (|) redirects the `stdout` from one command to `stdin` for another. The pipe is not considered a redirect operator but rather a control operator; while pipe is used to redirect output not all control operators have this function.

```console
 madflojo@netbook:~$ ls /tmp/ | grep filename
 filename
```

### Redirects stderr to stdout

`program > filename 2>&1`

When using redirects, you can use &N (where N is the file descriptor) to redirect 2 methods of output to the same path. The example below redirects stderr to the file; This can also be used with pipe (|) operator:

```bash
mayo@localhost:~> ls /not-exist 2>&1 | grep not-exist  
ls: cannot access '/not-exist': No such file or directory
```

You can also use the same redirection to suppress null error (redirects `stderr` to `/dev/null`)

```bash
mayo@localhost:~> ls /not-exist 2>/dev/null 
mayo@localhost:~> 
```


### The `tee` command

`tee` is neither a control or a redirect operator but is worth mentioning. `tee` is a command that will take the `stdin` given to it and write that both to the file specified and to `stdout`. It is useful when you both want to log the output and see the output on the terminal/screen.

```bash
mayo@localhost:~> cat ./not-exist 2>&1 | tee -a log.txt  
cat: ./not-exist: No such file or directory  
mayo@localhost:~> ls -la log.txt    
-rw-r--r-- 1 mayo mayo 44 Apr 10 06:22 log.txt  
mayo@localhost:~> cat log.txt    
cat: ./not-exist: No such file or directory
```

The `-a` flag append the output to the file specified rather than overwriting the entire file (pretty important for log files)

### Using pipe (|) and `grep`

`grep` is a useful command to find a specific matching string from a file or `stdout`. The `-v` flag gives the opposite results (shows lines that do not match that pattern)

```bash
mayo@localhost:~> cat data.txt    
hello hackers!  
hello world!  
mayo@localhost:~> cat data.txt | grep hackers  
hello hackers!  
mayo@localhost:~> cat data.txt | grep -v world  
hello hackers!  
mayo@localhost:~>
```

### `sed` command

Sometimes real data and garbage data are mixed in the same line, so a `grep` might not be sufficient. `sed` provides an easy way to substitute patterns in text with a different word. The syntax for matching and replacing is simple:

```bash
sed "s/oldword/newword/g"
```

`s/` - substitute  
`oldword` - the word to replace  
`newword` - the replacement for `oldword`  
`/g` - global (search for all occurrences of the pattern)
