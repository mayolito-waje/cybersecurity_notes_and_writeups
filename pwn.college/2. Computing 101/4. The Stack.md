### The stack

We learned how to store memory into registers. To step up, we will learn that there's one region of memory that we have access to in the program that need no setup from us: the **stack**.

The stack is a region of allocated memory used as a scratch space for our program, and the register `rsp` (*Stack Pointer*) points to the top of it. For now, let's start with this relevant detail: when a program starts, `rsp` points to data that represents the number of command-line arguments passed to the program (including the program name itself). For example, if you run:

````console
hacker@dojo:~$ /tmp/your-program hello world
````

Then the situation looks like this (actual addresses are an example):

````text
    Address    │ Contents
  +───────────────────────+
  │ ...        │ ...      │
  +───────────────────────+
┌▸│ 1337000    │ 3        │  ◀── the argument count
| +───────────────────────+
| | 1337008    | ???      |
| +───────────────────────+
| | 1337016    | ???      |
│ +───────────────────────+
│
└────────────────────────────┐
                             │
   Register │ Contents       │
  +────────────────────────+ │
  │ rsp     │ 1337000      │─┘
  +────────────────────────+
````

`rsp` references stack, and the value is `3`: one for program name, one for argument `hello`, and one for argument `world`. The stack also has other data shown, but we won't worry about that for now.

### Stack offsets

Recall that stack is a collection of data. This means that to access those other data, we can *offset* them!

To add offset when accessing `rsp`, we can use `[rsp+N]` where `N` is the number of bytes to offset. `[rsp+8]` reads 8-byte value right after `[rsp]`, `[rsp+16]` reads the next one after that, and so on.

````text
    Address    | Contents
  +---------------------------+
  | rsp        | value 0      |  <-- [rsp]
  +---------------------------+
  | rsp+8      | value 1      |  <-- [rsp+8]
  +---------------------------+
  | rsp+16     | value 2      |  <-- [rsp+16]
  +---------------------------+
  | ...        | ...          |
  +---------------------------+
````

For convention, offsets tend to be multiples of 8. That is because many values on the stack, such as number, memory address, and so on tend to be 8 bytes (64 bits). But at its core, the stack, like any other region of memory, is a contiguous region of individual bytes.

### Program arguments on the stack

We've now know that `[rsp]` read the argument count, and `[rsp+128]` gets data at an offset. Right after argument count, the stack stores *pointers* to each program argument. This means that `[rsp+16]` doesn't contain the argument text directly -- it contains the *address* where that text lives.

For example, if you run `/tmp/your-program Hi`:

````text
     Register │ Contents
   +───────────────────────────+
   │ rsp      │ 1337000        │─┐
   +───────────────────────────+ │
                                 │
  ┌──────────────────────────────┘
  │
  │    Address    │ Contents
  │  +────────────────────────+
  │  │ ...        │ ...       │
  │  +────────────────────────+
  └▸ │ 1337000    │ 2         │  ◀── the ARGument Count (termed "argc")
     +────────────────────────+
     │ 1337008    │ 1234000   │──────┐
     +────────────────────────+      │
     │ 1337016    │ 1234560   │────┐ │
     +────────────────────────+    │ │
     │ 1337024    │ 0         │    │ │
     +────────────────────────+    │ │
                                   │ │
   ┌───────────────────────────────┘ │
   │                                 │
   │   Address   │ Contents          │
   │ +──────────────────────────+    │
   │ │ 1234000   │ "/tmp/..."   │◀───┘ the program name
   │ +──────────────────────────+
   │ │ ...       │ ...          │
   │ +──────────────────────────+
   └▸│ 1234560   │ "Hi"         │ the first argument!
     +──────────────────────────+
````

So to get the actual data of the argument, you need to dereference twice: once to get the pointer from the stack, and once to follow it to the data.

````asm
mov rdi, [rsp+16] # load the first argument pointer (e.g. 1234000) from the stack
mov rdi, [rdi]    # follow the pointer to read the actual data (e.g. "Hi")
````

### Popping from the stack

Why is the stack called a *stack*? If you know computer science, a stack is a *stack*! You `push` values (which is added on top of the stack), then `pop` values (where you retrieve that last pushed value on the stack).

They're a little different though. For example, `pop rdi` does two things:

1. Reads the value at `[rsp]` into `rdi` (just like `mov rdi, [rsp]`).
1. Adds 8 to `rsp`, advancing the stack pointer to the next value.

So, popping and pushing from the stack involves manipulating the top address pointer for next value addition or retrieval.

Using the same example as before:

````console
hacker@dojo:~$ /tmp/your-program hello world
````

Before the `pop rdi`:

````text
    Address    │ Contents
  +───────────────────────+
  │ ...        │ ...      │
  +───────────────────────+
┌▸│ 1337000    │ 3        │  ◀── the argument count
│ +───────────────────────+
│ | 1337008    | ???      |
│ +───────────────────────+
│
└────────────────────────────┐
                             │
   Register │ Contents       │
  +────────────────────────+ │
  │ rsp     │ 1337000      │─┘
  +────────────────────────+
  │ rdi     │ 0            │
````

After the `pop rdi`:

````text
    Address    │ Contents
  +───────────────────────+
  │ ...        │ ...      │
  +───────────────────────+
  │ 1337000    │ 3        │
  +───────────────────────+
┌▸| 1337008    | ???      |
│ +───────────────────────+
│
└────────────────────────────┐
                             │
   Register │ Contents       │
  +────────────────────────+ │
  │ rsp     │ 1337008      │─┘
  +────────────────────────+
  │ rdi     │ 3            │
  +────────────────────────+
````

The value `3` is popped off the stack into `rdi`, and `rsp` advanced by 8 bytes to point to the next value! Notice that data `1337000` is still in memory, but as far as stack is concerned, it's been removed: `rsp` moved past it.
