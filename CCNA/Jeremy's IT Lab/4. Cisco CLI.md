### How to connect to a Cisco device? (Console port)

To connect a cisco device to configure it through the CLI, you must connect them first to the interface/console port.

![487](../../_assets/Pasted%20image%2020260427085434.png)

Let's say we want to connect the laptop to the Cisco device through RJ45 port. You will mostly need the cable below to connect to the port:

![287](../../_assets/Pasted%20image%2020260427085603.png)

The image above is a **rollover cable**. Notice that the one end has an **RJ45** connector, but the other end is a **DB9** connector. However, most modern laptops do not have DB9 connector by default, so you might need an **adapter** to connect to USB port on your laptop.

The rollover is similar to UTP cable, both having 8 pins. The connection pins works similar to crossover cable but with some difference:

![519](../../_assets/Pasted%20image%2020260427090020.png)

### PuTTy

Once you connect the computer to a Cisco device, how to access the CLI? You will probably use a terminal emulator, such as **PuTTy**. To access the CLI, you mus configure PuTTy to use *serial* connection.

![Pasted image 20260427085139.png](../../_assets/Pasted%20image%2020260427085139.png)

For serial connection, the default configs are:

* Speed (baud) set to 9600 bits per second
* Data bits set to 8
* Stop bits set to 1 (for each 8 bits of data, 1 stop bit is sent to mark the end of 8 bits)
* No parity (to detect errors)
* No Flow control (control the flow of data from transmitter to receiver)

Most of the time, you don't have to change the default settings for the serial connection. But it's nice to remember them.

After setting up PuTTy, you will be greeted with the terminal:

![Pasted image 20260427090537.png](../../_assets/Pasted%20image%2020260427090537.png)

### User EXEC mode

When first entering CLI, you will by default be in User EXEC mode

![279](../../_assets/Pasted%20image%2020260427090640.png)

User EXEC mode is very limited. Users can look at some things, but *can't* make any changes to the configuration. User EXEC mode is also known simply as "user mode".

### Privileged EXEC Mode

To enter, privileged EXEC mode, use `enable` command. And you will be meet with something like this:

![286](../../_assets/Pasted%20image%2020260427090928.png)

Privileged exec mode provides complete access to view the device's configuration, restart the device, etc. However, you still *cannot* change the configuration. But you *can* change the time on the device, the config file, etc.

**Comparison of available commands between user exec mode and privileged exec mode (limited to Packet Tracer, beyond Packet Tracer/CCNA, there are more commands available):**

![Pasted image 20260427091132.png](../../_assets/Pasted%20image%2020260427091132.png)

To view available commands, use a question mark (`?`).

### Global Configuration Mode

To enter global config mode, enter `configure terminal`

![Pasted image 20260427091931.png](../../_assets/Pasted%20image%2020260427091931.png)

You can also use shortcut command: `conf t`

![Pasted image 20260427092008.png](../../_assets/Pasted%20image%2020260427092008.png)

If you allow anyone to change the config, you risk your system and confidential files. So it is often recommended to set up a password. While in global configuration mode, use `enable password` command to enable password:

![Pasted image 20260427092215.png](../../_assets/Pasted%20image%2020260427092215.png)

Use `?` to check if there are additional commands you can utilize. `<cr>` means there are no more available commands, and you just have to press `Return` to execute the command.

Let's test our password, type `exit` to return to privileged exec mode. Enter `exit` again  to log out of privileged exec mode to return to user exec mode.

![Pasted image 20260427092712.png](../../_assets/Pasted%20image%2020260427092712.png)

Now, once you are in user exec mode, if you type `enable`, you will be meet with password authentication!

![Pasted image 20260427092810.png](../../_assets/Pasted%20image%2020260427092810.png)

For security purposes, the password does **not** display as you type it. Entering wrong password 3 times will deny you access because of bad secrets:

![Pasted image 20260427092938.png](../../_assets/Pasted%20image%2020260427092938.png)

### running-config / startup-config

There are two separate configuration files kept on the device at once.

* **Running-config** - the current, active configuration file on the device. As you enter CLI commands, you edit the active configuration.
* **Startup-config** - the configuration file that will be loaded upon restarting the device.

To view the running configuration file, use `show running-config` while in privileged exec mode:

![Pasted image 20260427093556.png](../../_assets/Pasted%20image%2020260427093556.png)

To view the startup config, use `show startup-config` while in privileged exec mode:

![Pasted image 20260427093748.png](../../_assets/Pasted%20image%2020260427093748.png)

The message shows "startup-config is not present". This is because we haven't saved the running configuration yet. So every time we restart the router, it will load up the default configuration, not the startup configuration.

To actually save the running config, you can use either of the three commands, executed from privileged exec mode:

* `write`
* `write memory`
* `copy running-config startup-config`

![Pasted image 20260427094059.png](../../_assets/Pasted%20image%2020260427094059.png)

Upon saving the running configuration, if you type `show startup-config` again, you will actually have some data:

![Pasted image 20260427095109.png](../../_assets/Pasted%20image%2020260427095109.png)

You've been noticing (if you are security pilled) that the password is *exposed* in plain text! This is dangerous if it got in the wrong hands.

To mitigate such risk, you can use Cisco's proprietary encryption algorithm with the `service password-encryption` command. You have to be in global configuration mode first to enter the command:

![Pasted image 20260427095909.png](../../_assets/Pasted%20image%2020260427095909.png)

If you look at config again, noticed that password is now encrypted:

![Pasted image 20260427095946.png](../../_assets/Pasted%20image%2020260427095946.png)

The number `7` before the encrypted password means it used Cisco's proprietary encryption algorithm serviced using `service password-encryption`.

`service password-encryption` is better than not using any encryption algorithm. But it's still not secure. You might be able to crack the *ciphertext* if you used some cracking tool through google search:

![Pasted image 20260427100322.png](../../_assets/Pasted%20image%2020260427100322.png)

You can use a secret using `enable secret <password>` command. Run `do sh run` to run `show running-config` while in global configuration mode to view the running configuration. The `do` command lets you run Privileged EXEC mode commands while in global configuration mode. `sh run` is shortcut for `show running-config`.

![Pasted image 20260427100522.png](../../_assets/Pasted%20image%2020260427100522.png)

Now we have both a secret and a password. The password still used the same encryption method. But it will be *ignored* if you enabled the secret!  The `enable password` is still there for backward compatibility and security transitions. So even if you changed the password with `service password-encryption`, it won't have an effect to the secret and you still have to enter the secret to authenticate. Use secret for better authentication!

The `5` before the encrypted secret means it uses **MD5** encryption which is a stronger encryption method. It can still be cracked since no password is truly invincible (although modern encryption methods might take you billions of years to crack them through brute force).

### Cancelling commands

The `no` command cancels the previously entered commands. This effectively negates the protocols and commands you previously set in any type of modes.

![Pasted image 20260427102114.png](../../_assets/Pasted%20image%2020260427102114.png)

The `no service password-encryption` command didn't decrypt the already saved password in configuration file. But if you change the password next time, it will be saved as a plain text.
