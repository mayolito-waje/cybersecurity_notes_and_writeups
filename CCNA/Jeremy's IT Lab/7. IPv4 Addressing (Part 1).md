### What is the `/24` in `192.168.1.254/24`?

![Pasted image 20260430101145.png](../_assets/Pasted%20image%2020260430101145.png)

You might have noticed along the way what does `/24` in IP pertains to. It just simply means the first 24 bits of the IP address *represents* the **network portion of the address**, and the remaining 8 bits represents the **end host**! So the first 24 bits of the IPv4 is the first 3 octets or 3 bytes (8 + 8 + 8 = 24), and the last byte is the end host. This means that `110000001010100000000001` or `192.168.1` is the network portion, and `11111110` or `254` is the end host!

![Pasted image 20260430101715.png](../_assets/Pasted%20image%2020260430101715.png)

Notice that for each LAN, the network portion of the IP addresses are the same (`192.168.1.X/24` and `192.168.2.X/24`), and only the end hosts differ. So, a single local network carry the same value of network portion in IP address across all connected devices.

Now let's try with a different type of IP Address: `154.78.111.32/16`:

![Pasted image 20260430102449.png](../_assets/Pasted%20image%2020260430102449.png)

The `/16` part means the first 16 bits of the address is the network portion, or the first 2 bytes. So `154.78` is the network and `111.32` is the host!

![Pasted image 20260430103021.png](../_assets/Pasted%20image%2020260430103021.png)

As last practice, in `12.128.251.23/8`, the `12` is the network and `128.251.23` is the host!

### IPv4 Address Classes

![Pasted image 20260430103653.png](../_assets/Pasted%20image%2020260430103653.png)

As you can see in the table, class D and E are reserved for multicast addresses and reserved/experimental addresses respectively. Notice that in class A, instead of 0-127 first octet numeric range, its *0-126*. Why?

### Loopback Address

That is because the `127` value on first octet is reserved for **loopback addresses**, which includes an IP of `127.0.0.0` up to `127.255.255.255`! The loopback address is used to test the 'network stack' (think OSI, TCP/IP model) on the local device.

![Pasted image 20260430104259.png](../_assets/Pasted%20image%2020260430104259.png)
![Pasted image 20260430104334.png](../_assets/Pasted%20image%2020260430104334.png)

If you `ping` to a loopback (in this case, `127.0.0.1` and `127.23.68.241`) address, the computer will respond back to its own ping! Notice the **round trip time** (RTT), all 0 ms! This is because the traffic isn't going anywhere, the PC is just sending and receiving traffic in and of itself.

### Netmask

![Pasted image 20260430110233.png](../_assets/Pasted%20image%2020260430110233.png)

The `/X` is actually a modern way of writing the prefix length of a network address (such as in Juniper devices). However, Cisco devices still uses the older way of addressing prefix length, also known as **dotted decimal netmask**. A netmask is written like an IP address, but the network portion is all 1s and the host portion is all 0s, as exampled in the case above. So, the prefix length and the netmask are actually the same thing, just written in different ways!

### Network Address

![Pasted image 20260430111223.png](../_assets/Pasted%20image%2020260430111223.png)

If the host portion of the address is all 0s, this is the **Network Address**, indicating the network itself. So, in the example above, the IP address `192.168.1.0/24` is the network itself, so other devices cannot use it. That is why PC1 IP address is `192.168.1.1/24` since it is the first usable address.

### Broadcast Address

![Pasted image 20260430111650.png](../_assets/Pasted%20image%2020260430111650.png)

The last address of the network, which is the host portion containing all 1s, is the **Broadcast Address**. Like the network address, the broadcast address CANNOT be assigned to a host. This left us with 254 usable address in a Class C IP address, since `192.168.1.0/24` and `192.168.1.255/24` is reserved for network and broadcast address, respectively. Noticed that the router used the last usable IP address over a LAN, which is `192.168.1.254/24`!

So, the broadcast IP address is the address to send the packet to all hosts of the local network. So, if a packet with Destination IP of `192.168.1.255` is sent over a local network, what would be the destination MAC of the Ethernet frame after the IP address is encapsulated? The destination MAC is the *broadcast MAC Address!*

![Pasted image 20260430112416.png](../_assets/Pasted%20image%2020260430112416.png)

So if PC1 sends a packet containing destination IP of broadcast address, it will be received by PC2 and R1's G0/0 interface.
