### Variable-Length Subnet Masks

* Until now, we have practiced subnetting used FLSM (Fixed-Length Subnet Masks).
* This means that all of the subnets use the same prefix length (ie. subnetting a class C network into 4 subnets using /26).
* VLSM (Variable-Length Subnet Masks) is the process of creating subnets of different sizes, to make your use of network addresses more efficient. VLSM is more complicated than FLSM, but it's easy if you follow the steps correctly.

![Pasted image 20260519100359.png](../_assets/Pasted%20image%2020260519100359.png)

* If we use FLSM, we have to borrow 3 bits to accomodate for 5 needed subnets. But the host portion will be left only with 5 bits which will fit a maximum number 30 usable addresses. Tokyo LAN A have 110 hosts and Toronto LAN B have 45 hosts so it won't fit the available hosts per subnet if we use FLSM.

### VLSM - Steps

1. Assign the largest subnet at the start of the address space.
1. Assign the second-largest subnet after it.
1. Repeat the process until all subnets have been assigned.
