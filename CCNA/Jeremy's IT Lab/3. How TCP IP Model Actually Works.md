### Protocols and Standards

A **protocol** is a set of rules defining how data should be communicated between devices over a network. The protocols are the "languages" that computers use to communicate.

Since early days of computer networking, there have been several attempts to define functions needed for computers to communicate with each other.

![632](../../_assets/Pasted%20image%2020260426082738.png)

With proprietary approach, communications between different vendors was difficult since they mostly made some protocols to be used with their own products.

A **standard** is an agreed-upon specification that describes how a protocol or technology should work. **Vendor-neutral** standards allow for devices of all types to communicate with each other. For example, an Apple MacBook can access a website hosted on a web server running Linux.

### Who define the standards?

Networking standards/protocols are not developed by a single vendor, but by independent standards organizations with participation from engineers at many companies.

![Pasted image 20260426083431.png](../../_assets/Pasted%20image%2020260426083431.png)

**IEEE (Institute of Electrical and Electronics Engineers)**

* Developed many of the technologies used on local networks such as **Ethernet (802.3)** and **Wi-Fi (802.11)**

![517](../../_assets/Pasted%20image%2020260426083539.png)

**IETF (Internet Engineering Task Force)**

* Open community that defines protocols used on the Internet such **TCP, IP, UDP, HTTP, DNS, etc.**
* Publishes standards in documents called **RFCs** (Requests for  Comments)

### Layered models

![558](../../_assets/Pasted%20image%2020260426084007.png)

Networks do a lot of different jobs to move data from one computer to another. It can be physical transmission of signals, local delivery through LAN, routing traffic between networks, end-to-end conversations, etc.

With **layered models**, it lets us group *related jobs*  into layers. So, each layer has a specific role. One thing to keep in mind that layers **uses** the service of the layer *below* and **provides** services to the layer *above*. Protocols, live *mostly* at one layer. For example, TCP and UDP lives at the **transport** layer. Together, they form a **stack** of protocols that work as a team (the **network stack**).

Keep in mind though, that layered models are a *description, not a law*. Different textbooks/courses use slightly different models (4-layer, 5-layer, etc.).

### The TCP/IP Model

![Pasted image 20260426085205.png](../../_assets/Pasted%20image%2020260426085205.png)

**Application Layer** - Application layer are protocols for communication between application processes; create and interpret the data.

![564](../../_assets/Pasted%20image%2020260426085345.png)

**Transport Layer** - Transport layer provides end-to-end communication between application processes using **port numbers**.

![588](../../_assets/Pasted%20image%2020260426085455.png)

**Internet Layer** - Internet layer provides end-to-end communication between hosts across the network using **IP Addresses** and routers.

![597](../../_assets/Pasted%20image%2020260426085704.png)

**Local Network Layer** - The Local Network layer provides hop-to-hop delivery within a local network using **MAC addresses** and switches

![601](../../_assets/Pasted%20image%2020260426085857.png)

**Physical Layer** - Sends bits as electrical, optical, or radio signals over the physical medium.

### Encapsulation and Decapsulation

**Encapsulation**

![Pasted image 20260426151853.png](../../_assets/Pasted%20image%2020260426151853.png)

* The **Application layer** prepares the data to be sent over the network.
* As message moves down the stack, each layer **encapsulates** the data with a **header**, that includes information needed for that layer. For example, source and destination address in the form of port numbers, IP addresses, MAC addresses, etc. are added as information within the layer.
* Layer 2 also adds **trailer** that the receiving device uses to check for transmission errors.
* The **Physical layer** transmits the bits as signals over the physical medium. **L2 header** is transmitted first, and the **L2 trailer** is transmitted last.

**Decapsulation**

![Pasted image 20260426152311.png](../../_assets/Pasted%20image%2020260426152311.png)

* The receiving device receives the message as a stream of bits at Layer 1.
* The device examines the information in Layer 2 header and trailer, and then removes them (**decapsulation**).
* The decapsulation process continues up the stack: Layer 3 removes L3 header, then Layer 4 removes L4 header, and then the data is delivered to the Application layer.
* The application process the data and generates a response that goes back down the stack if needed.

### Protocol data units

At each stage of encapsulation/decapsulation process, there is a name given to the message. These messages also have alternative names to describe them at each stage in TCP/IP layer: **protocol data unit (PDU)**.

Each PDU's contents *before* adding the layer's header are called a **payload**. So, everything encapsulated by that layer's header/trailer is the payload.

* **Layer 4 PDU (L4PDU)**
  
  ![362](../../_assets/Pasted%20image%2020260426152952.png)
  
  In layer 4, the combination of data and a L4 header is called a **segment** (TCP) or **datagram** (UDP), which is the layer 4 PDU. The segment or datagram's *payload* is the **application data**.

* **Layer 3 PDU (L3PDU)**
  
  ![386](../../_assets/Pasted%20image%2020260426153355.png)
  
  In layer 3, the PDU is the combination of a segment/datagram and an L3 header, which is called a **packet**. In this layer, the **segment/datagram** is the *payload*.

* **Layer 2 PDU (L2PDU)**
  
  ![423](../../_assets/Pasted%20image%2020260426153631.png)
  
  In layer 2, the PDU is the combination of a packet and L2 header/trailer, which is called a **frame**. The frame is what is actually sent *over the wire*. In frame, the *payload* is the **packet**.

### Adjacent-layer interaction

![661](../../_assets/Pasted%20image%2020260426154434.png)

In TCP/IP, each layer provides a service to the layer *above* it, and is serviced by layer *below* it. This is called an **adjacent-layer interaction**.

* Layer 4 services Layer 5 by delivering data to the correct application using port numbers.
* Layer 3 services Layer 4 by delivering segments/datagrams to the correct destination host using IP addresses.
* Layer 2 services Layer 3 by delivering packets to the next hop using MAC addresses.
* Layer 1 services Layer 2 by sending and receiving frames as electrical, optical, or radio signals.

### Same-layer interaction

![651](../../_assets/Pasted%20image%2020260426154743.png)

Each layer also communicates with the same layer on other devices. This is called a **same-layer interaction**.

* Application layer on one host sends data to the Application layer on other host.
* A segment/datagram is addressed to the **Layer 4 port number** of the correct application on the destination host.
* A packet is addressed to the **Layer 3 IP address** of the destination host.
* A frame is addressed to the **Layer 2 MAC address** of the next hop.
* Signals sent out of a physical port are received by a **physical port** on the connected device.

### The OSI model

* The TCP/IP development started in 1970s (ARPANET work, early TCP/IP specs)
* In late 1970s and 1980s, the **International Organization for Standardization (ISO)** designed a 7-layer **Open Systems Interconnection (OSI) model** and a matching protocol suite. The goal was to create international, vendor-neutral networking standards that could unify existing proprietary stacks and potentially replace TCP/IP.
* Governments and the US promoted OSI as the preferred stack for new deployments.
* However, OSI protocols ended up being late and complex, and never gained the same deployment as TCP/IP.
* Some OSI technologies are still used, but TCP/IP still "won" in the real world.
* Today, almost all networks use TCP/IP, but the OSI model survives as a reference/teaching model and a common way to talk about "layers".
* The TCP/IP Application layer is often called **Layer 7** because of the reference to the OSI model (even though TCP/IP model is mostly 5 layers).
