### First/Last Usable Address

![557](../_assets/Pasted%20image%2020260501092718.png)

Remember that `00000000` and `11111111` are reserved for network and broadcast address, respectively. To get the first usable address, you have to add 1 to the network address, so `00000000 + 1` is `00000001`. To get the last usable address, you have to subtract 1 from the broadcast address, so `11111111 - 1` is `11111110`. So, the host portion of `00000001` is the first usable address and the host portion of `11111110` is the last usable address.

## Cisco Commands for IPv4 Addressing

![Pasted image 20260501093344.png](../_assets/Pasted%20image%2020260501093344.png)

Here's a small network for demonstration put together using GNS3. There are 3 LANs connected to a single router `R1`. Keep in mind the diagram above is not realistic. In real world setting, a LAN will be composed of multiple devices with multiple switches.

### `show ip interface brief`

![Pasted image 20260501094608.png](../_assets/Pasted%20image%2020260501094608.png)

You have to be in privileged EXEC mode to use the above command. To be in privileged EXEC mode, you have to use `enable` or `en` command to enter. `show ip interface brief` is a command to confirm the status of each interface on the device as well as their IP addresses.

* **Interface** - list the available interfaces
* **IP-Address** - list the IP addresses of the interfaces. They are not assigned as of the moment.
* **OK?** - Check whether if the IP address is valid or not. This is mostly a legacy feature and not that important. Modern devices won't let you assign invalid IP addresses so you will most likely never find NO value in this column. Unassigned IP is a valid IP address so the value is still YES.
* **Method** - method by which the interface was assigned an IP address. It is currently unset.
* **Status** - **layer 1** status of interface. It currently shows `administratively down` which means that the interface has been disabled with the `shutdown` command. `administratively down` is actually the default status of Cisco *router* interfaces. However, Cisco *switch* interfaces are NOT `administratively down` by default (either up if connected to another device, or down if not connected).
* **Protocol** - **layer 2** status of the interface. Because the interfaces at layer 1 are in shutdown, layer 2 cannot operate, so it shows `down` state. We will never see `down` status in `Status` column and `up` in `Protocol` column, although the reverse is possible.

### IPv4 Address configuration

![Pasted image 20260501100002.png](../_assets/Pasted%20image%2020260501100002.png)

Once you are in global configuration mode, you can configure IP addresses on an interface using `interface interface-name` command. Notice `gigabitethernet 0/0` and `gigabitethernet0/0`, they are both valid in Cisco CLI and refers to the same interface.

After typing the command, we will enter **interface configuration** mode as you can see in `(config-if)#` cue before the command input.

![468](../_assets/Pasted%20image%2020260501100212.png)

For lazy folks, the shortest command for `interface` is `in`, though `int`, `inte`, `inter`, up to `interface` will also work.

![464](../_assets/Pasted%20image%2020260501100437.png)

Notice that `g` alone have a lot of equivalent commands, so `g` alone won't work to refer to `gigabitethernet`. But if you add `0/0` to `g`, the CLI will understand it! Cool.

![Pasted image 20260501100815.png](../_assets/Pasted%20image%2020260501100815.png)

Now that we are in interface config mode, we can now set the IP address to that interface. We can use the command `ip address <ip-address>`. We can use **context-sensitive help** (the `?` after the command) to look for extra options.

In the command above, we set the IP address for `GigabitEthernet0/0` (which is the router `R1` in the diagram) with `10.255.255.254` and a subnet mask of `255.0.0.0`. Remember that the IP address we set is a *class A* IP address, which have a prefix of `/8`, so it would have a subnet mask of `255.0.0.0`!

After setting the IP address and subnet mask, there is still some job to do. As with the previous note, the router interface is currently in shutdown, since it is a Cisco router which is has a `shutdown` command enabled by default (compared to Cisco switch). To enable that interface, enter the command `no shutdown`. In previous lesson, we learned that `no` before a command cancels that enabled command!

![Pasted image 20260501101957.png](../_assets/Pasted%20image%2020260501101957.png)

If you look at status of the interfaces once again, notice that the interface `GigabitEthernet0/0` now have an assigned IP address of `10.255.255.254`. Neat! The `Method` column is also set to `manual` (since we manually configured it). The `Status` and `Protocol` are now `up`!

### `show interfaces command`

![600](../_assets/Pasted%20image%2020260501102800.png)

While you can simply only use `show interfaces`, the terminal can be clogged with bunch of information, so it is more preferable to specify interface to check with `show interface <interface>` to only show information about that interface.

This command shows primarily the detailed Layer 1 and Layer 2 information about the interface, with also some information on Layer 3.

### `show interfaces description`

![Pasted image 20260501103234.png](../_assets/Pasted%20image%2020260501103234.png)

With `show interfaces description`, the output is notably similar to `show ip interface brief`, but with extra `Description` column. This is completely optional but provides benefits by adding extra information about that interface and explain the purpose of each interface.
