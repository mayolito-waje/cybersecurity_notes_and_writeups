**Ethernet** - ethernet is a collection of network protocols/standards.

***Why do we need network protocols/standards?***

![Pasted image 20260425164150.png](../../_assets/Pasted%20image%2020260425164150.png)
Think of a *lingua franca*. We often talk to foreigners in a language we can both understand (such as English). This is how network standards works in a similar fashion. For example, Ethernet is the agreed standard when connecting cables to access the network.

### Bits and Bytes

Connection between devices in a network, operate at a set speed, which is often in *bits per second* (Kbps, Mbps, Gbps, etc), *not* bytes per second. A bit is representation of either 0 or 1 (which computer understands). 8 bits equals to 1 byte.

### UTP Cables

**UTP** - means **Unshielded Twisted Pairs**. This is the copper cable used in Ethernet standards.
![405](../../_assets/Pasted%20image%2020260425165449.png)

Four pair of wires are *twisted* together to protect against EMI (Electromagnetic Interference)

![654](../../_assets/Pasted%20image%2020260425165634.png)

**RJ-45** connectors, as you can see on the picture above, have **8 pins**. But not all network devices use all pins. Let's start with how does Ethernet and Fast Ethernet cabling connects to these pins.

**UTP cabling for 10BASE-T, 100BASE-T**

![568](../../_assets/Pasted%20image%2020260425170916.png)
![570](../../_assets/Pasted%20image%2020260425165949.png)
**Full duplex** - allows for both devices to send and receive data at the same time without collision. As you can see on the diagram above, the router sends data to switch through pins 1 and 2, while it can receive data from the switch through pins 3 and 6. The same principle is used when connecting computers and switch.

The diagram above demonstrates **straight-through cabling**. Pins 1 and 2 connects to pins 1 and 2 on the other end, respectively and pins 3 and 6 from one end connects to pins 3 and 6 to other ends respectively.

We don't always connect routers to switches, or PCs to switches. What if we want to connect from router to router? Or from PC to another PC? Let's try to connect them with straight through cabling:

![Pasted image 20260425171014.png](../../_assets/Pasted%20image%2020260425171014.png)

Both routers will transmit on pins 1 and 2, and receive data on pins 3 and 6. So it won't work!

We need cross connection for such problem, which is also called as **crossover cable**.

![Pasted image 20260425171430.png](../../_assets/Pasted%20image%2020260425171430.png)
![Pasted image 20260425171518.png](../../_assets/Pasted%20image%2020260425171518.png)

Since both the router and a PC transmits on pin 1 and 2, and receives data on pin 3 and 6, crossover cabling is used for UTP cabling.

### Auto MDI-X

In the old days, we have straight-through and crossover cables to handle collisions when sending and receiving data between devices through Ethernet standards. But most modern network devices include a feature called **Auto MDI-X**. Auto MDI-X automatically detect which pins the neighbor devices transmits and receive data on, and allow to adjust.

![658](../../_assets/Pasted%20image%2020260425172237.png)

### UTP Cabling for 1000BASE-T and 10GBASE-T

Like mentioned above, Gigabit and 10Gig Ethernet uses all 8 pins in RJ-45 cables.

![Pasted image 20260425172535.png](../../_assets/Pasted%20image%2020260425172535.png)

Unlike 10BASE-T and 100BASE-T, each pair of pins in 1000BASE-T and 10GBASE-T is **bidirectional**, meaning they are not specifically dedicated to either sending or transmitting data, so they work *both ways (either transmitting or receiving)*, allowing them to operate at much faster speeds.

### Fiber-Optic Connections

![Pasted image 20260425173029.png](../../_assets/Pasted%20image%2020260425173029.png)

Copper UTP wiring can extend up to 100 meters, which is plenty within the LAN. But how about for larger areas? In the router images above, the boxed ports in yellow outline is where you insert RJ-45 connectors (which uses copper UTP cables). How about orange-boxed interfaces?

![320](../../_assets/Pasted%20image%2020260425174139.png)

This is what you will insert into those interfaces, it is called **SFP Transceiver (Small Form-Factor Pluggable)**. You connect these to interfaces, but still not complete. You need cables.

![642](../../_assets/Pasted%20image%2020260425174424.png)

This is called a **fiber-optic cable**. Rather than electrical signals over copper wiring (such as in UTP cables), these cable send *lights over glass fibers*. Notice that there are two connectors on each end, one is for sending data, and one is for receiving data, on each end.

![Pasted image 20260425175127.png](../../_assets/Pasted%20image%2020260425175127.png)

Fiber optic cables have two wiring, because each wiring is one way direction for transmitting data like the example diagram above.

**Structure of fiber optic cable**

![Pasted image 20260425175646.png](../../_assets/Pasted%20image%2020260425175646.png)

There are two main types of fiber optic cables: **single-mode fiber** and **multi-mode fiber**.

#### Multi-mode fiber

![Pasted image 20260425175917.png](../../_assets/Pasted%20image%2020260425175917.png)

* Core diameter is wider than single-mode fiber.
* Allow multiple angles (modes) of light waves to enter the fiberglass core
* Allows longer cables than UTP, but shorter cables than single-mode fiber.
* Cheaper than single-mode (due to cheaper LED-based SFP transmitters)

#### Single-mode fiber

![Pasted image 20260425180125.png](../../_assets/Pasted%20image%2020260425180125.png)

* Core diameter is narrower than multimode fiber.
* Light enters at a single angle (mode) from a laser-based transmitter\]
* Allows longer cables than both UTP and multimode fiber
* More expensive than multimode fiber (due to more expensive laser-based SFP transmitters)
