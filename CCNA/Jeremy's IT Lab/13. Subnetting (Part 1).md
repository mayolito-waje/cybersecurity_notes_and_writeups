### IANA

![Pasted image 20260516103651.png](../_assets/Pasted%20image%2020260516103651.png)

* The IANA (Internet Assigned Numbers Authority) assigns IPv4 addresses/networks to companies based on their size.
* For example, a very large company might receive a **class A** or **class B** network, while a small company might receive a **class C** network.
* However, this led to many wasted IP addresses

**Why many wasted IP addresses?**

![Pasted image 20260516103919.png](../_assets/Pasted%20image%2020260516103919.png)

* There are total of 3 networks in the diagram above, the network on the left side, the center, and the right side. Both San Francisco (R1) and New York (R2) are connected to 2 networks.
* The network where San Francisco and New York are connected through a network address of `203.0.113.0/24` is called a **point-to-point network**. A point-to-point network is a connection of 2 nodes/points.
* In `203.0.113.0/24` address, there are total of 4 address used out of 256 available addresses: the network address (`203.0.113.0`), broadcast address (`203.0.113.255`), San Francisco address (`203.0.113.1`), and New York address (`203.0.113.2`). The *other 252 addresses* are **wasted**.

**Another case of wasted IP addresses**

* *Company X* is a big company, and will need IP addressing for 5000 end hosts.
* A class C network does not provide enough addresses (only 254 available addresses), so a class B network must be assigned.
* However, this will result in about *60000 wasted IP addresses* from about 65000 available IP addresses in class B.

### CIDR (Classless Inter-Domain Routing)

* When the Internet was first created, the creators did not predict that the Internet would become as large as it is today.
* This resulted in wasted address space like the examples mentioned (there are many more examples).
* 4 billion available addresses are not a problem of the past. But IPv4 address exhaustion is a problem today.
* The **IETF (Intenet Engineering Task Force)** introduced **CIDR** in 1993 to replace the 'classful' addressing system.
* With CIDR, the requirements of Class A having /8 prefix, Class B having /16 prefix, and Class C having /24 prefix were removed. This allowed larger networks to be split into smaller networks, allowing greater efficiency. These smaller networks are called 'subnetworks' or 'subnets'.

**No CIDR**
![474](../_assets/Pasted%20image%2020260516105926.png)

**CIDR Practice!**

With CIDR, the number of host bits are flexible and is not hardcoded based on IPv4 class. So instead of 2<sup>8</sup> - 2 for number of usable address, the formula would be **2<sup>n</sup> - 2**.

* `203.0.113.0/25` = 126 usable addresses
* `203.0.113.0/26` = 62 usable addresses
* `203.0.113.0/27` = 30 usable addresses
* `203.0.113.0/28` = 14 usable addresses
* `203.0.113.0/29` = 6 usable addresses
* `203.0.113.0/30` = 2 usable addresses

**But how about /31**?

In normal IPv4, we reserve the first and last available host for network and broadcast address. In CIDR with /30 netmask, we will be left with *0 usable address* (2<sup>1</sup> - 2 = 0). But for a point-to-point network, it's no problem! We can use /31 mask *as long as it is point-to-point network only*. A dedicated connection between 2 routers don't actually need network and broadcast address.

Take note that Cisco routers will still warn you if you use /31 mask, but as long as you do not configure it on non point-to-point interface, you can just ignore it!

### Subnetting Scenario

![651](../_assets/Pasted%20image%2020260516114208.png)

* 47 (45 hosts + 1 broadcast address + 1 network address) multiplied by 4 is **188** available hosts. So Class C IP address is no problem.
* We need 4 equal size subnets with enough room for **45** hosts

![589](../_assets/Pasted%20image%2020260516114954.png)

/27 mask only has 30 usable addresses, not enough to fill a room for one subnet which holds 45 hosts. Using **/26** mask is enough to fill in room for 45 hosts. So we will use /26 netmask! /26 mask still have more than available address than the needed 45 hosts. Unfortunately, we cannot pick a subnet that fill exactly the number of hosts that we need, there will be some wasted address. This is fine because there might be some more room to grow anyway if we decided to add more hosts in a subnet.
