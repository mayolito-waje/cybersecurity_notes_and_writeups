If you were to somehow get inside the network cable and see what was going back and forth over that connection, you would see there was a lot of activities going on across many different systems. Some operations include:

* accessing important data from another part of the world
* remote access for secure network communication (such as screen sharing)
* managing traffic and prioritizing important applications
* adhere to protocols to maintain uptime and availability

### Content Delivery Network (CDN)

![Pasted image 20260421111040.png](../../_assets/Pasted%20image%2020260421111040.png)

**Content Delivery Network (CDN)** is designed for efficient retrieval of data from one central point to the end user. CDN is often setup in geographical areas and users are able to cache information in each one of these CDN locations depending on proximity of user for faster and more efficient retrieval. For example, if you reside in North America, you will often retrieve data from North America server instead of retrieving from central CDN that resides on the opposite end of the planet.

If you are watching a video on YouTube, you are retrieving data from CDN! Because CDN allows for distribution of data across different servers in the world so end users can access these assets faster and efficiently without them knowing.

### Virtual Private Network (VPN)

![Pasted image 20260421111727.png](../../_assets/Pasted%20image%2020260421111727.png)

If you value privacy, VPN can be your friend. VPN allows for secure communication because the data being sent over the network traverses through an encrypted tunnel. VPN allows for encrypted communication on an insecure medium.

![Pasted image 20260421112651.png](../../_assets/Pasted%20image%2020260421112651.png)

VPN often used a device called a **concentrator/head-end** device to be the central connection point for all users to access that VPN service. A concentrator is purpose built appliance designed for high speed encryption and decryption of network data in real-time (send a message, encrypt that data, traverse through the network, decrypt that data, then end-user receives it and vice versa). While can be a standalone device, we often integrate concentrators within a **Next Generation Firewall (NGFW)** itself.

You might sometimes not need a purpose built appliance for VPN concentrator when doing some small business operation. Many VPN concentrators run as software on an existing Operating System and is shipped and available to popular OS such as Linux, Windows, MacOS, etc.

### Quality of Service (QoS)

Not all applications are equal and run simultaneously on a network, some are higher priority that need immediate operation on an organization. For example, there is an ongoing real-time video stream which is higher priority than say file transfer.

![Pasted image 20260421113048.png](../../_assets/Pasted%20image%2020260421113048.png)

Quality of Service (QoS) offer functionality to manage prioritization of services/applications. You might also refer to QoS as traffic shaping or packet shaping. This will allow admins to control what applications can flow through the network, usually based on bandwidth usage or data rates. This might allow real time applications to have greater control or access to the network and put lower priority on applications that may be transferring files (prioritization).

To manage QoS, you may need to apply some changes to configuration to routers, switches, firewalls, or QoS devices.

### Time To Live (TTL)

![Pasted image 20260421114322.png](../../_assets/Pasted%20image%2020260421114322.png)

A task can infinitely run through a loop without any human intervention. We might need to intervene when a task is running too long and when should we stop the tasks from running over a network. We invented **Time to live (TTL)** for such purpose.

A TTL is a "timer" for how long a data should be available. For example, in packet traversal, TTL serves as a limitation on how many "hops" can a packet traverse over the network before it drops and not reach the destination. TTL can also be used before a website cache becomes deleted by setting up a duration on how long can that cache live.

### Some different uses of TTLs

**Routing loops**

![Pasted image 20260421114549.png](../../_assets/Pasted%20image%2020260421114549.png)

When a packet traverses over the network and reaches Router A, Router A might thinks the next hop is Router B, and Router B might thinks the next hop is Router A, effectively creating a loop.

Routing loops occur when you have many routers and those routers have many different routes. So it is easy to misconfigure IP in a static route and create routing loops over the network. By setting a TTL inside the IP packet itself, the loop will continue until the TTL reaches zero and drop the packet!

**IP (Internet Protocol)**

Since loops could cause a packet to live forever, we always set a hop limit before the packet drops to prevent such issues. In most cases ,for **each pass of an IP packet in a router**, the TTL *decreases* by one until it reaches zero, effectively ending the loop.

* In macOS/Linux, the default TTL for an IP packet is **64 hops**
* In Windows, the default TTL for an IP packet is **128 hops**

**DNS (Domain Name System)**

![Pasted image 20260421115659.png](../../_assets/Pasted%20image%2020260421115659.png)

A TTL in DNS context pertains to how long does our local devices *caches* the DNS lookup (using `dig` or `nslookup`) for a certain amount of time. In this case, it is **TTL seconds** long. Based on the picture above, the value 300 means "keep this IP address in your cache for 300 seconds". After 300 seconds passed, our local cache removes the particular DNS resolution.
