There are many different ways to forward network traffic. Networking devices forward traffic over the network and every device serves purpose. New technologies are added all the time and old techs are being consistently deprecated. The said can be same for networking technologies.

### Router

**Router** routes traffic between IP subnets. Router is an OSI layer 3 device and is sometimes called "layer 3 switches". Router often connects diverse network types such as LAN or WAN which can be connected through copper-based connection, fiber-based connection, and other things.

### Switch

Switch is a layer 2 device because it forwards traffic based on destination MAC address. In switches, bridging is usually done in hardware (Application-specific integrated circuit or ASIC). Switches contain many ports and features and is mostly the core of enterprise network. Switches may also provide Power over Ethernet (PoE) connection. If a switch is a multilayer or provide routing functionality, it can be also be called "Layer 3 Switch".

![Pasted image 20260419223605.png](../../_assets/Pasted%20image%2020260419223605.png)

### Firewalls

By simplest implementation, firewall filter traffic by TCP/UDP port number. Firewalls can be traditional firewall or Next Generation Firewall (NGFW) which is more modern and is capable of identifying applications traversing over network. Firewalls also encrypt traffic such as setting up VPN tunnel between sites. Some firewalls can also act as a router (Layer 3 device) since it often sits on the ingress and egress of the network, provide Network Address Translation (NAT), or allow dynamic routing.

### IDS and IPS

Intrusion Detection System / Intrusion Prevention System watches network traffic and mitigate possible exploits against operating systems, applications, etc. IDS and IPS can catch intrusions buffer overflows, cross-site scripting, and other vulnerabilities. IDS alarm or alert the user if it detects intrusion while IPS stop it before it gets into the network.

### Load Balancers

![Pasted image 20260419224746.png](../../_assets/Pasted%20image%2020260419224746.png)

Load balancers distribute traffic loads across multiple servers to handle traffic and allow for continuous operation without bottleneck. LB is often invisible to the end-user. Load balancers are common in large-scale implementations such as web server farms, database farms, etc. Load balancers often have high fault tolerance so it can quickly respond to server outages through updating server rotation and offer fast convergence.

![Pasted image 20260419224849.png](../../_assets/Pasted%20image%2020260419224849.png)

Aside from traffic distribution, Load Balancers also often give the following services:

* TCP offloading (protocol overhead)
* SSL offloading (encryption/decryption)
* Caching for faster response from the server
* Prioritizing traffic through Quality of Service (QoS)
* Content switching (application-centric balancing) by having dedicated server for one type of program/service

### Proxies

![Pasted image 20260419225535.png](../../_assets/Pasted%20image%2020260419225535.png)

Proxies sits between the users and the external network. The purpose of proxy is to receive user requests and sends the request on their behalf to the server and vice versa. This is helpful for mitigating possible malicious software sent by the server or end-user. Proxies are useful for caching information, access control, URL filtering, content scanning, etc. Some applications may need to know how to use the proxy (explicit) but some requires no manual configuration (transparent).

### NAS vs. SAN

It is very common to store files on centralized storage facility on data centers. Network Attached Storage (NAS) connects to a shared storage device across the network. NAS provides file-level access which can be inefficient (reads the whole file and overwrite the whole file when proposing changes)

Storage Area Network (SAN) is more efficient as it provides block-level access which is very efficient for reading and writing files. Instead of copying the entire file, we can just extract the important parts (blocks) of the file to read and modify.

Both NAS and SAN requires a lot of bandwidth because we often interact with them when reading or modifying large number of files. It is common for both SAN and NAS to have their own high-speed isolated networks.

### Access point (AP)

![Pasted image 20260419231017.png](../../_assets/Pasted%20image%2020260419231017.png)

AP allows us to connect our device wirelessly to the rest of the network. AP is not a router. A wireless router is a router, access point, and a switch in a single device. AP is common in enterprise buildings and serve only one purpose: wireless communication. AP is a bridge because it extends the wired Ethernet network onto the wireless network. AP is an OSI layer 2 device.

In most cases, a building mostly do not have only one AP and is mostly available in number of areas in the building. An access point may not even be in the same building. Within a single AP, we might need to setup access and security policies, adjust AP configs, etc.

### Wireless LAN controllers

![Pasted image 20260419231829.png](../../_assets/Pasted%20image%2020260419231829.png)

Instead of connecting to a single access point to adjust configs, access policies, etc. We can use Wireless LAN controllers for centralized management of access points. We can deploy new access points, monitor their security and performance, configure them and deploy changes to all sites, read AP usage reports, etc.

Wireless LAN controllers are often proprietary and is paired with available access points when buying them.
