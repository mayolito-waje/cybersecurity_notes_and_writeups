
### OSI model

- Open Systems Interconnection Reference Model
- It is a guide, thus the term "model"
- This is not the OSI protocol suite (most of OSI protocols didn't catch on)
- There are unique protocols at every layer

**Layer 7: Application**

- The layer we see on the screen. Anytime we interact with the app, we interact with layer 7
- Common protocols in application layer include HTTP, FTP, DNS, POP3

**Layer 6: Presentation**

- Setting the standard format, character encoding, application encryption/decryption, etc.
- Often combined with the application layer
- **Application encryption (SSL/TLS)**

**Layer 5: Session**

- To send and receive data, a communication must be established. This is where the session layer works by initializing and controlling communication management between devices
- **control protocols, tunneling protocols**

**Layer 4: Transport**

- The "post office" layer
- Uses Transmission Control Protocol (TCP) and User Datagram Protocol (UDP) to transport IP packets
- **TCP Segment, UDP datagram**

**Layer 3: Network**

- The "routing" layer
- Handles traffic over the network
- While the data link layer looks for destination MAC address, the network layer looks for the Internet Protocol (IP) address
- fragments frames to traverse different networks
- **IP Adress, router, packet**

**Layer 2: Data Link**

- The basic network "language"
- Also known as the MAC (Media Access Control) layer, MAC address is the most common associated with the network cards in the Data Link Control (DLC) protocol
- MAC Address/DLC Address is the unique physical address of every adapter cards 
- Also known as the "switching" layer because we use MAC Address to determine the location we want to sent the data to through the network
- **Frame, MAC address, Extended Unique Identifier (EUI-48, EUI-64), Switch**

**Layer 1: Physical**

- Describe the physical signals sent through through cables or fibers on the network
- The physics of network (signaling, cabling, connectors)
- You have a physical layer problem - a bad cabling, punch-downs, etc. run loopback tests, test/replace cables, swap adapter cards
- **cables, fiber, and the signal itself**


To summarize:

**Physical:** Electrical signals -> **Data Link:** Ethernet connection -> **Network:** Encapsulate IP -> **Transport:** Encapsulate TCP -> **Session:** Link the presentation to the transport -> **Presentation:** SSL encryption -> **Application:** https://mail.google.com
